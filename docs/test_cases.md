### TC-01: Создание задачи с корректными данными

**Предусловия**:  
- В текущей директории отсутствует файл tasks.json (или он пуст/не существует).  
- Модуль запускается из командной строки.

**Шаги**:
1. Запустить программу: python src/app.py
2. Ввести команду создания задачи (например, `add "Купить молоко" "1 литр"`)
3. Ввести команду просмотра списка задач (`list`)

**Ожидаемый результат**:  
- В списке отображается одна задача с названием «Купить молоко», описанием «1 литр», статусом по умолчанию (например, «новая») и непустым идентификатором.  
- После завершения программы и повторного запуска задача снова загружается из файла.

**Постусловия**:  
- Файл tasks.json существует и содержит корректную JSON-запись о созданной задаче.

### TC-02: Попытка создания задачи с пустым названием

**Предусловия**:  
- Модуль запущен.

**Шаги**:
1. Ввести команду создания задачи с пустым названием, например: add "" "описание"
2. Проверить, появилось ли сообщение об ошибке.
3. Ввести команду list

**Ожидаемый результат**:  
- Модуль выводит сообщение об ошибке (например, «Название задачи не может быть пустым»).  
- Задача не создаётся, список задач остаётся без изменений.
