# Чек-лист регрессионного тестирования

- Запуск программы без ошибок (команда python src/app.py или аналогичная)
- Создание задачи с корректными данными (название и описание)
- Просмотр списка задач (созданная задача отображается)
- Изменение статуса задачи на допустимый (например, с «новая» на «в работе»)
- Сохранение задач в JSON и загрузка после перезапуска (проверить, что данные не потеряны)
- Поиск по ключевому слову (находит задачу, если слово есть в названии или описании)
- **Ошибочный сценарий**: попытка создать задачу с пустым названием (должна быть ошибка, задача не создаётся)
- **Ошибочный сценарий**: запуск при повреждённом JSON-файле (модуль не падает, начинает с пустого списка)
